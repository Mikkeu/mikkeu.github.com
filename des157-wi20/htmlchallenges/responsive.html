About Media Queries

Media queries are used in html to have certain CSS only work if certain conditions are true. It is very speciifc. It is useful for changing a page's layout if the page changes sizes like it is being used for different monitors. so if the monitor is less than a certain amount of pixels, it will change so all the information can be displayed well. Min width is if the screen is a certain number or more then it is true. Max width is if the screen is a certain number but smaller then it is true. When using media queries, it is standard to design mobile first. This is because it will load faster for smaller devices than if we worked in reverse, because there is less for the mobile device to load and change.

Relative Units

A relative unit is a sizing option that gets it's size from a relative length of something else. For example, em is a font relative sizing. So if the font size changes so will the sizing of the elements using this sizing method. This is good to use for screens that change size and for the fonts that change size with it. If you don't do this the spacing will change disproportionally with the rest of the elements on the page. Percentage is a bit different. If one element that is not contained in another is set to 90% width then it is 90% of how much the user can see. If there is an element inside that element that is set to 90% width as well then it is 90% of the element is it in not of the space of the viewport.

Meta Viewport Tag

The viewport is the part of the browser we can see. The meta viewport was added to control the viewport's size and scale. This is useful if the monitor is too small and the webpage does not fit it. When I checked my webpages after taking off the meta viewport tag, I noticed it was a lot slower when adjusting for changing the window browser size. With the meta tag it was much smoother and easier to use if the page kept changing.
